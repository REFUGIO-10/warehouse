# `scraped/` — rival intelligence (RAMA C)

Snapshot of the **public** REFUGIO leaderboard. The platform publishes every team's
result **and source code** with no auth, so we mirror it here for other agents to study
and beat. Reading public solutions is part of the intended meta (see `../AGENTS.md`).

> Generated by `../tools/scrape.py`. Treat everything here as **regenerable output**:
> don't hand-edit `jobs.json` / `standings.json` / `snapshot.json` / `INDEX.md` /
> `solutions/*.py` — re-run the scraper instead. This `README.md` is the only file here
> that is written by hand.

## What's here

| File | What it is |
|---|---|
| `INDEX.md` | Human/agent-readable standings + table of downloaded solutions (best first). **Start here.** |
| `snapshot.json` | Scrape time, frontier (max deliveries seen), our best, and job ids new since last run. |
| `standings.json` | Per-team best deliveries / best points (derived). |
| `jobs.json` | Every job's metadata: status, points, deliveries, runtime, timestamps, per-run breakdown (`--details`), safety review. |
| `solutions/NNNN__<team>__<job>.py` | Each succeeded job's **full source**, `NNNN` = zero-padded deliveries so files sort best-first. Each file has a metadata header; the body is the rival's verbatim `create_layout()` + `act()`. |

## Refresh the snapshot

From the repo root (`warehouse/`):

```bash
python tools/scrape.py            # refresh metadata + download any new solutions
python tools/scrape.py --details  # also pull per-run blocked_moves / remaining_distance / safety text
python tools/scrape.py --force    # re-download all source even if already saved
```

Pure stdlib — no installs, no browser. Idempotent; it reports which jobs are new.

**Auto-refresh:** once PR #2 is merged to `main`, `.github/workflows/scrape.yml`
re-runs the scraper every ~15 min and commits any changes here, so this folder
stays current on its own. You can also trigger it manually from the Actions tab
(`workflow_dispatch`).

## How other agents should use this (the whole point)

1. **Pick a target** from `INDEX.md` — the highest-delivery rivals above our best.
2. **Re-score it in our oracle** to get an apples-to-apples number on our seeds:
   ```bash
   python tools/benchmark.py scraped/solutions/<file> --count 20
   ```
   Expect bench projection ≈ official deliveries (small gap = public vs hidden seeds).
3. **Diff the geometry vs the routing.** Each solution is layout (`create_layout`) +
   policy (`act`). Figure out *which* gives them the edge:
   - Copy their `create_layout()` into `submissions/layout_dev.py` (RAMA A), keep our
     `act`, and benchmark — isolates the layout's contribution.
   - Copy their `act()` into `submissions/policy_dev.py` (RAMA B), keep a baseline
     layout, and benchmark — isolates the policy's contribution.
4. **Beat it, then integrate.** A new variant only goes into `submissions/submission.py`
   once the bench (`--count 20+`) beats the live frontier. Then validate
   (`python refugio-starter-kit/tools/check_submission.py submissions/submission.py`)
   and hand to the submit operator (`submissions/SUBMITS.md`).

> ⚠️ Several top solutions use `numpy` (and may use `scipy`/`numba`). To re-score those,
> install the allowed packages in the kit venv:
> `pip install -e refugio-starter-kit -r refugio-starter-kit/requirements.txt`.
> Stdlib-only rivals run with a plain `python3`.

## Notes / caveats

- **Public seeds ≠ hidden seeds.** Bench numbers are estimates; rank by them, don't
  treat them as the official score.
- **Deliveries ≠ points.** Points reward pushing the frontier *first* — some teams have
  high deliveries but 0 points (they matched late). To earn points we must re-pass the
  current frontier in `snapshot.json`.
- Don't commit secrets/tokens; this scraper only reads public pages.
